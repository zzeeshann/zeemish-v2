# Preface

I built Zeemish mostly with an AI assistant over a few weeks. Then I launched it. Then I tried to explain to someone what it actually does, and I realised I couldn't. Not properly. I could point at the dashboard and say "13 agents." I could point at the pieces and say "one a day, automatically." But if they asked *how*, I'd start a sentence and then stop.

That is a bad way to own a thing.

This book is what I wrote so that when someone asks me how Zeemish works, I can hand them something. And so that the next time I open the code six months from now, I can read this instead of staring at file names I don't remember writing.

It's written for someone like me — someone who knows a little about programming but not a lot, who has heard words like "Cloudflare" and "agent" and "database" and has a vague sense of what they mean but couldn't explain them at a dinner party. It assumes you've used a computer and maybe visited GitHub once. It doesn't assume anything beyond that.

If you already know what a database is, skip the database chapter. If you've shipped code to Cloudflare Workers before, skip the Cloudflare chapter. The book is modular on purpose.

The voice of this book is the same voice Zeemish uses when it publishes its daily pieces. Plain English. Short sentences. No flattery. No "unlock your potential." If I translate a technical word, I translate it in the same sentence I use it. If a concept has ten common misconceptions attached to it, I tell you about the two that actually matter.

One more thing. I built this as an honest project. The README says honest software, no seams hidden. The dashboard shows you every piece's audit history. The decision log explains why each piece of the architecture is the way it is. The followups log admits what's still broken. This book is the fourth surface where that honesty lives. I don't pretend the 13 agents are a team of independent minds. I don't pretend the system was "learning" before I closed the loop. I tell you what the thing is.

If you find a mistake, the repo is public. Open an issue. Corrections are cheap and welcome.

— Zishan
